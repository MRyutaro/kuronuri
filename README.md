# kuronuri

PDF を読み込んで、隠したい場所をドラッグで指定し、
黒塗り済みの PDF として保存できる、GitHub Pages 向けの静的 Web アプリです。
処理はブラウザ内だけで完結し、黒塗りした部分の文字は保存後の PDF に残りません。

## できること

- PDF をブラウザ内で読み込む
- ページをプレビューしながら、ドラッグで黒塗り範囲を追加する
- ページごとに黒塗りを削除・クリアする
- 黒塗り済み PDF をダウンロードする

## 黒塗り方式

このアプリは、見た目だけ黒い矩形を重ねるのではなく、保存時に MuPDF.js の redaction API を使います。

- 指定範囲のテキストを PDF から削除する
- 覆われた画像ピクセルを処理する
- 覆われた線画も処理する

つまり、出力は「PDF のまま」で、かつ単なる見た目上の塗りつぶしではありません。

## 構成

ビルド不要の静的ファイルだけで動きます。

- `index.html`
- `styles.css`
- `app.js`
- `vendor/mupdf/`
- `.nojekyll`

`pdf.js` は CDN から読み込み、`vendor/mupdf/` には公式 `mupdf@1.27.0` の配布物を同梱しています。

## ローカルで試す

単純な静的配信で動きます。

```bash
python3 -m http.server 8000
```

その後、ブラウザで `http://localhost:8000` を開いてください。

## GitHub Pages で公開する

1. この内容を `main` ブランチに push する
2. GitHub の `Settings` を開く
3. `Pages` を開く
4. `Build and deployment` で `Deploy from a branch` を選ぶ
5. Branch を `main`、folder を `/ (root)` にする
6. 保存する

数分後に GitHub Pages の URL でそのまま動きます。

## 使い方

1. PDF をアップロードする
2. プレビュー上をドラッグして黒塗り範囲を作る
3. 必要に応じてページ移動や削除を行う
4. `黒塗り PDF を保存` を押す

## 制約

- 元文書がスキャン画像だけの PDF の場合、削除対象はテキストではなく画像ピクセルになります
- 複雑な PDF では、ビューアごとの差や特殊な描画命令の影響が残ることがあります

## ライセンス注意

同梱している MuPDF.js は公式 `mupdf@1.27.0` で、`AGPL-3.0-or-later` です。
商用利用や配布条件に制約が出る場合は、Artifex の商用ライセンスを確認してください。

## 補足

- すべての処理はブラウザ内で完結します
- 大きな PDF は redaction と保存に時間がかかることがあります
